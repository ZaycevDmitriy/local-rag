# Рабочий процесс разработки (AI Factory)

AI Factory работает в двух фазах: **конфигурация** (выполняется один раз на проект) и **рабочий цикл разработки** (повторяемый цикл из исследования, планирования, доработки, реализации, верификации, коммитов и эволюции).

## Фаза конфигурации проекта

При начальной настройке `ai-factory init` создаёт контекстные файлы, от которых зависят все последующие команды. Сюда входит создание базовых документов (`DESCRIPTION.md`, `AGENTS.md`), настройка скиллов и MCP-интеграций, а также опциональная настройка архитектурной документации, правил, дорожных карт и конфигураций деплоя.

## Цикл разработки

Основной повторяемый процесс следует такой последовательности:

**Исследование (опционально)**: `/aif-explore` — изучение идей и сравнение подходов перед фиксацией плана. Результаты можно сохранить в `.ai-factory/RESEARCH.md` для переноса между сессиями.

**Гейт достоверности**: Когда критически важна точность — `/aif-grounded` обеспечивает «ответы только на основе фактов» и отклоняет предположения.

**Планирование**: `/aif-plan fast` — быстрый план без ветвления; `/aif-plan full` — создаёт git-ветку для крупных фич.

**Доработка**: `/aif-improve` — второй проход анализа для выявления пропущенных задач и исправления зависимостей.

**Реализация**: `/aif-implement` — выполнение запланированных задач с контрольными точками коммитов.

**Верификация**: `/aif-verify` — проверка соответствия кода плану, архитектуре, дорожной карте и правилам.

**Коммит**: `/aif-commit` — генерация conventional commits с валидацией через context-gate.

**Эволюция**: `/aif-evolve` — извлечение уроков из исправлений и улучшение контекста скиллов для будущих запусков.

## Справочник команд

| Команда | Назначение | Артефакты |
|---------|-----------|-----------|
| `/aif-explore` | Исследование и анализ компромиссов | Опционально `.ai-factory/RESEARCH.md` |
| `/aif-plan fast` | Быстрые задачи без ветвления | `.ai-factory/PLAN.md` |
| `/aif-plan full` | Полноценные фичи с git-ветками | `.ai-factory/plans/<branch>.md` |
| `/aif-implement` | Выполнение запланированных задач | Код + контрольные точки коммитов |
| `/aif-verify` | Проверка качества и соответствия | Только чтение, валидация |
| `/aif-fix` | Расследование и исправление багов | `.ai-factory/patches/` |

Такая архитектура обеспечивает «предсказуемые результаты» за счёт выполнения по плану, а не свободного исследования — это позволяет возобновлять сессии и предотвращает расползание скоупа.
