### Reject `console.log` in MCP-reachable directories
**Source**: 2026-04-16-22.25.md
**Rule**: Flag any diff that introduces `console.log(` under `src/search/**`, `src/mcp/**`, `src/storage/**`, `src/embeddings/**`. For shared files that contain both CLI-only and MCP-reachable methods (e.g., `src/storage/chunk-contents.ts`, `src/storage/chunks.ts`), trace the call graph: if the method is invoked from `SearchCoordinator.search` or any MCP tool handler, the logging site must use `console.error`. Verify that `MCP_REACHABLE_FILES` / `MCP_REACHABLE_METHODS` in `src/mcp/__tests__/stdout-discipline.test.ts` were updated if new MCP-reachable code was added.

### MCP API symmetry and inputSchema completeness
**Source**: 2026-04-16-23.58.md
**Rule**: For any MCP-tool PR, verify: (a) parameter names are consistent across tools that take the same entity — `search`, `read_source`, `list_sources` must all use `sourceName` (and never `sourceId` in one, `sourceName` in another); (b) every parameter shown in `CLAUDE.md` examples or `docs/` is registered in the Zod `inputSchema`. Zod silently drops unknown fields, so a documented-but-unregistered param is a live contract gap.

### MCP tools must emit `isError: true` on validation failure
**Source**: 2026-04-16-23.58.md
**Rule**: Verify that every MCP tool handler converts thrown `Error`s from the coordinator / storage into `{ isError: true, content: [{ text: '... error: ...' }] }`. Returning an empty successful payload on a validation failure is a bug — agents interpret silence as "no matches" and proceed. Look for `try/catch` blocks in `src/mcp/tools/*.ts` that swallow errors into empty arrays.

### Dry-run vs real path parity
**Source**: 2026-04-16-23.58.md
**Rule**: For commands with `--dry-run` (e.g., `export-cmd`) or interactive previews, verify that counts, file lists, sizes, and any other aggregate come from the same helper as the real execution path. Duplicate SQL or formulas in the preview branch diverge after schema migrations (classic symptom: `undefined фрагментов` / `NaN` in dry-run while real command works). Require a shared helper in `src/commands/_helpers/`.

### Backfill loops mark every row in every outcome branch
**Source**: 2026-04-17-16.01.md
**Rule**: For any PR that adds or modifies a backfill command (SELECT ... WHERE col IS NULL → UPDATE), verify that ok, skip, and fail branches all write a row marker. Persistent errors on the same rows without a sentinel turn the loop into a retry pulsemachine — catastrophic on paid LLM/API calls. Expected pattern: `[skipped:<reason>]` / `[failed:<reason>]` text sentinels paired with `embedding = NULL` so rows are excluded from vector search.

### Atomicity red flag: two consecutive UPDATEs on the same row
**Source**: 2026-04-17-16.01.md
**Rule**: Flag code that issues two sequential `UPDATE`s on the same row from a single CLI path (e.g., `updateSummaries` followed by `updateSummaryEmbeddings`). Require a combined API that writes all conceptually-paired columns in one transaction / single SQL statement, unless each intermediate state is independently consistent (rare). A crash between them leaves the row half-written and, for `WHERE col IS NULL` selectors, permanently invisible.

### Regression tests cover crash-recovery, not only happy path
**Source**: 2026-04-17-16.01.md
**Rule**: For storage / backfill PRs, require a regression test that simulates a process crash at the transactional boundary — for example, "entire batch failed" as proxy for "process died after summary write but before embedding write". A positive-only test ("happy path still works") is insufficient and provides no protection against the bug class that caused the patch.

### `commands/` stays thin — business logic belongs in `_helpers/`
**Source**: 2026-04-17-16.01.md, .ai-factory/RULES.md
**Rule**: If a `.action()` callback in `src/commands/*.ts` contains non-trivial loop / state / error-aggregation logic, request an extraction into `src/commands/_helpers/*.ts` with an explicit signature and unit-test file. `.action()` is untestable — business logic there is unreviewable. This is an explicit project rule (`RULES.md`: "commands/ и mcp/ должны оставаться тонкими адаптерами").
