### New Module Requires index.ts Barrel Export
**Source**: ARCHITECTURE.md, rules/base.md
**Rule**: When planning a new module under `src/`, ALWAYS include an explicit task to create `index.ts` with public API exports. Modules without `index.ts` barrel violate the project's module boundary rules — other modules can only import via `index.ts`. Task description should list which types/functions will be exported.

### File Naming Conventions
**Source**: rules/base.md
**Rule**: Plan tasks must specify exact filenames following project conventions: CLI command handlers → `*-cmd.ts` (e.g., `gc-cmd.ts`, `re-embed-cmd.ts`); MCP tool handlers → descriptive kebab-case in `src/mcp/tools/` (e.g., `read-source.ts`); implementations and utilities → kebab-case (e.g., `factory.ts`, `mcp-entry.ts`).
