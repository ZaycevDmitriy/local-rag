## Rules

### ESM Imports: Mandatory .js Extension
**Source**: codebase convention (src/cli.ts, src/mcp-entry.ts, rules/base.md)
**Rule**: ALWAYS use `.js` extension in all local import paths when writing or modifying TypeScript code. Example: `import { X } from './module/index.js'`, NOT `from './module/index'`. Missing `.js` extension causes runtime `ERR_MODULE_NOT_FOUND` errors in TypeScript ESM projects. This applies to all files including test files.

### Module Boundary: Only via index.ts Barrel
**Source**: ARCHITECTURE.md, RULES.md, rules/base.md
**Rule**: Import from other modules ONLY through their `index.ts` barrel export. Direct imports from internal module files are forbidden. Example: use `import { createDb } from '../storage/index.js'`, NOT `import { createDb } from '../storage/db.js'` (when writing code outside the storage module). This is a hard project rule — violations will be rejected in code review.

### MCP Stdout Constraint
**Source**: codebase convention (src/mcp-entry.ts comment, RULES.md)
**Rule**: In any code inside `src/mcp/`, use ONLY `console.error()` for logging. NEVER add `console.log()` to `src/mcp/` code — stdout in the MCP server is exclusively reserved for the MCP protocol and any `console.log()` will corrupt it. CLI code (`src/commands/`, `src/cli.ts`) uses `console.log()` for user-facing output and `console.error()` for errors.
